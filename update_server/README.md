# 更新服务器文件

此目录包含更新服务器的所有文件，需要上传到您的更新服务器。

## 文件说明

| 文件 | 说明 |
|------|------|
| `update_admin.php` | 管理后台（网页界面配置版本信息 + 安装统计） |
| `update_api.php` | API接口（供用户系统调用，自动记录安装信息） |
| `update_config.json` | 配置文件（自动生成，存储版本信息） |
| `install_stats.json` | 统计数据（自动生成，存储安装记录） |

## 功能特点

- 📊 **安装统计** - 自动记录所有安装用户信息
- 🌐 **域名追踪** - 记录每个安装的域名
- 📱 **版本监控** - 查看哪些用户使用最新版本
- 🔍 **搜索功能** - 快速搜索特定域名
- 📈 **统计面板** - 直观的数据展示

## 部署步骤

### 1. 上传文件

将整个 `update_server` 目录上传到您的服务器，例如：
```
https://update.your-server.com/
├── update_admin.php
├── update_api.php
├── update_config.json
└── install_stats.json (自动生成)
```

### 2. 设置权限

确保目录可写入：
```
chmod 755 update_server/
```

### 3. 访问管理后台

```
https://update.your-server.com/update_admin.php
```

默认密码: `admin123`（登录后请修改）

### 4. 配置用户系统

在用户系统的 `includes/config.php` 中配置：
```php
'system' => array(
    'version' => '1.0.0',
    'update_url' => 'https://update.your-server.com/update_api.php',
),
```

## 使用流程

```
1. 访问 update_admin.php 登录
2. 在"安装统计"标签页查看所有安装用户
3. 在"版本配置"标签页配置新版本
4. 上传更新包到服务器
5. 填写更新包下载地址
6. 保存配置
7. 用户在后台检查更新即可看到新版本
```

## 安装统计说明

当用户系统检查更新时，会自动记录以下信息：

- **域名** - 用户安装的网站域名
- **版本** - 当前使用的版本号
- **首次访问** - 第一次检查更新的时间
- **最后访问** - 最近一次检查更新的时间
- **检查次数** - 累计检查更新的次数
